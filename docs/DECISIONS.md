# Decisions log — The Academy

Resolutions for the `[DECISION]` flags in `SCOPE.md`, plus build-time choices.
Platform decisions inherited from The Seminar (models, embeddings, envelope
streaming, voice, portraits, EPUB handling) carry over unchanged and are
recorded in that repo's decisions log; the table below is Academy-specific.

| # | Decision | Choice | Rationale |
|---|---|---|---|
| A1 | First ingested text | `republic-jowett` (Gutenberg #1497, Jowett translation) | Anchors M1 (Elenchus on justice, *What Is Justice?*). Jowett died 1893; translation 1871/1894 — unambiguously PD-US. Pipeline confirmed: Jowett's long Introduction/Analysis is front matter and is excluded; Books I–X chapterize as ch 0–9, 510 passages |
| A2 | Plato translation | Jowett throughout MVP | Clear PD status, readable, and the standard free edition; Bloom/Grube/Reeve are NOT PD |
| A3 | Nietzsche translations | Thomas Common (*Zarathustra*, PG #1998), Helen Zimmern (*Beyond Good and Evil*, PG #4363), Ludovici — all pre-1930 ✅. **Kaufmann is NOT public domain — never ingest** | Per scope `[DECISION]` flag; recorded in LICENSING.md |
| A4 | New session kinds | `elenchus`, `thoughtExperiment`, `argumentLab` in migration 0004; `dialogue`, `rediscovery` reserved for V1 migrations | Matches MVP cut in scope §4; keeps kind check-constraint churn to one migration per tier |
| A5 | Commitment writes | Dual-path: in-turn envelope `commitmentOps` (professor notices live) + post-session MODEL_LIGHT extraction sweep (catches what the professor missed); in-turn ops win on conflict | In-turn-only misses positions asserted mid-flow; batch-only loses the professor's judgment about what was *actually asserted* vs. floated |
| A6 | Tension computation | Direct `conflicts` edges + 1-hop entailment (`A entails X, X conflicts B`), no deeper closure | Conservative by contract (scope §7): false contradictions are the feature's failure mode; depth-2+ chains compound ontology judgment calls |
| A7 | Tension surfacing threshold | Both sides `asserted` with `affirm_count ≥ 2`; at most one tension in the digest; oldest-unraised first | Scope guardrail (minimum evidence); one-per-session keeps it a scalpel, not a nag |
| A8 | Ontology storage | Authored `content/ontology/claims.json`, seeded to `claims`/`claim_edges` catalog tables; versioned like personas | It's content, not code — human-reviewable, diffable, and the validator cross-checks course `relatedClaims` against it |
| A9 | Ontology starter size | ~60 claims (≥10 per domain) for M2, growing toward ~200 | Scope build order M2 says start ~60; the required-id list in CONTRACTS §12.6 keeps course authoring unblocked |
| A10 | Thought-experiment branching | Authored nodes render client-side with no API call; the professor enters only at interrogation/debrief | Determinism + zero-latency branching + screenshot-shareable cards; the LLM interrogates the *why*, which is where it adds value |
| A11 | Argument-map rendering | Deterministic SwiftUI layered layout from `ArgumentSpec` (no LLM in the render path) | Scope §5 names this a primitive; correctness of the diagram must not depend on model output |
| A12 | iOS rebrand | Directory + target `TheAcademy`, display name "The Academy", bundle id `com.theacademy.app` | Clean fork; XcodeGen makes the rename cheap |
| A13 | Elenchus completion rule | `completeSession` rejected unless `phase == "reflection"`; professor must `declareOutcome` (aporia or robust) by revision 4 | Enforces the aporia-ends-in-reflection guardrail mechanically, not just in the persona doc |
| A14 | Worldview page vs. Profile page | Worldview replaces the reader-profile page as the Academy's identity surface; the reader-profile *pipeline* stays on (it still tunes teaching) but its UI is folded into Worldview | One identity surface; commitments are the product here, attention dimensions are supporting cast |
| A15 | Standalone sessions (E-M1) | `sessions.enrollment_id` nullable + `user_id`/`persona_id` columns (migration 0005), not a fake "commons" course | Honest data model, no phantom catalog rows; the cost (a branch in session loading, RLS arm) is one-time; relationship memory skipped for standalone sessions at MVP |
| A16 | Daily-question scheduling | Deterministic rotation: bank sorted by id, index = daysSinceEpoch(client local date) % size; no cron, no server schedule state | Client and server independently compute the same question; offline local notifications can carry the question text; re-encounters (E-M3) fall out of the same arithmetic a cycle later |
| A17 | One-tap commitment strength | Server writes the tapped option's mapped claim deterministically at `lean` via the §12.2 fold; `assert` only via model commitmentOps grounded in the typed sentence | A tap is a signal, not a conviction; keeps the map's evidentiary bar intact while letting the ritual accrete positions daily |
| A18 | Clinic map construction | The model builds the user's argument via stateOps (`setConclusion`/`addPremise`/`revisePremise`/`markCrux`) into the same spec shape the deterministic renderer consumes; render path stays LLM-free | Reuses the A11 renderer untouched; server-side op validation (id namespace, supports-refs, cap 8) keeps the map well-formed regardless of model behavior |
| A19 | Clinic guardrails + no RAG | argumentClinic and dailyQuestion skip retrieval at MVP (citations empty by contract); clinic never renders verdicts on who's right, never referees relationships; therapy-adjacent material → name the limit, point at the human step | The clinic's value is structure extraction on the user's material — corpus quotes add latency without value here; the guardrail line is what makes "philosophy applied to your life" shippable |
| A20 | Changelog storage | Append-only `commitment_events` ledger written by the fold (service role only); changelog and the changed-my-mind stat are derived, never stored; affirm events kept in the table, filtered in UI | History can't be reconstructed from overwritten rows; deriving keeps one source of truth; owner contest cascades events away with the commitment — the changelog records the examined life, not bookkeeping |
| A21 | Drops reuse thoughtExperiment | Weekly drops run the existing kind as standalone sessions with the spec from a `drops` catalog; selection = weeksSinceEpoch(local date) % bank, same arithmetic family as A16 | Zero new state machines; the authored-nodes/interrogation/debrief pipeline and iOS card rendering are reused untouched; re-encounters (E-M3) fall out of the week counter |
| A22 | Crowd aggregate privacy | SECURITY DEFINER RPC, gated on the caller's own response existing, returns first-choice distribution only, suppressed below 10 responses | Aggregates-after-answering is the §14.6 ethic mechanically enforced; first-choice-only keeps the payload anonymous and the reveal simple; suppression prevents small-crowd deanonymization |
| A23 | Steelman rubric | Four named levels (strawman / sketch / competent / signable) scored via `recordSteelmanScore` stateOp after ≤2 probe rounds; ladder = max level per target, computed client-side | Named levels grade the argument, not the person; the probe cap prevents endless revision loops; storing raw scores and deriving the ladder keeps the server dumb |
| A24 | Tension reconciliation | `markTensionReconciled` bound server-side to the single tension the session's digest raised; abandoning a side dissolves rather than reconciles | The model never sees tension ids, so binding to the raised one is the only sound reference; keeping dissolve ≠ reconcile preserves the celebration's meaning |
| A25 | News brief prefetch | MODEL_LIGHT + web_search generates ONE brief per week, cached in `news_briefs` keyed by weeksSinceEpoch; the seminar model never searches | Keeps the envelope/streaming contract intact; one search per week per deployment; a shared community question beats per-user drift |
| A26 | News even-handedness | Authored lens pairs (`content/news/lenses.json`, ≥1 per domain); both lenses get a full session phase; declining a position is a named legitimate outcome; no aggregates on news questions ever | "Even-handed by construction" must be structural, not a prompt hope; the lens pair is authored content so the opposition is real, not model-improvised |
| A27 | Practice shape | One `practice` kind with three modes (morning/evening/visualization) + a weekly `practiceReview` with a 7-day entries digest; entries unique per (user, mode, local date); no mood tracking, no practice scores | Micro-sessions match the ritual's size; the digest gives Bede receipts without surveillance framing; scores would turn practice into performance — the anti-goal |
| A28 | Bede at E-M3 | Author the persona (+ practice specs) now; *The Stoic Gymnasium* course stays V1 | The Practice Wing needs its professor, not his whole course; persona-first matches how the wing is actually used |
| A29 | Re-encounters | A repeat response to the same drop in a later week IS the re-encounter; no new tables; compare view derives from the user's own `drop_responses` | The (user, drop, week) key already encodes the arc; storage-free means the feature can't drift from the data |
